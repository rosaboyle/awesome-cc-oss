# Contributing Guidelines

Thank you for your interest in contributing to **Awesome Claude Code OSS**! Please read and follow these guidelines before submitting a pull request.

By participating in this project, you agree to abide by the [Code of Conduct](#code-of-conduct).

---

## How to Contribute

### Adding a New Resource

1. Fork this repository
2. Create a new branch: `git checkout -b add-new-resource`
3. Make your changes to `README.md`
4. Commit your changes: `git commit -am 'Add new resource'`
5. Push to the branch: `git push origin add-new-resource`
6. Submit a pull request

### Updating Your Pull Request

If a maintainer asks you to edit your PR, simply amend your commit and force push:

```bash
git commit --amend
git push --force
```

---

## Guidelines for Entries

Please ensure your pull request adheres to the following:

- **Search previous suggestions** before making a new one to avoid duplicates.
- **Use the following format:** `- [**Name**](link) — Description`
- Descriptions should be **clear, concise, and non-promotional**.
- Descriptions should start with a capital letter and end without a period (unless multiple sentences).
- New categories or improvements to the existing categorization are welcome.
- Keep descriptions **short and simple**, but descriptive.
- Check your **spelling and grammar**.
- Make sure your text editor is set to remove trailing whitespace.
- The pull request should have a **useful title** and include a link to the resource and why it should be included.

### What Qualifies as "Awesome"?

This is a curated list, not a comprehensive one. Submissions should be:

- **Directly related** to the Claude Code source code leak incident (March 31, 2026)
- **High quality** — well-written articles, insightful analysis, significant community discussions, or notable source code archives
- **Publicly accessible** — all links must point to publicly available content
- **Not spam or self-promotion** — resources should provide genuine value to the community

### Categories

Place your entry in the appropriate section:

- **Source Code Archives** — Repositories hosting or mirroring the leaked source
- **Clean-Room Reimplementations** — Independent rewrites inspired by the leak
- **Technical Breakdowns & Analysis** — Detailed technical deep dives
- **Key Discoveries from the Leak** — Notable findings from the source
- **News Coverage** — Mainstream media and tech press articles
- **Expert & Community Reactions** — Tweets, Reddit threads, HN discussions
- **Blog Posts & Deep Dives** — Independent blog posts and analyses
- **Videos & Podcasts** — Multimedia content
- **Security & Legal Implications** — Legal analysis, DMCA discussion, security concerns
- **Related Prior Incidents** — Historical context
- **Further Reading** — Additional reference material

---

## Quality Standards

- Only include resources that are **genuinely useful** and provide value
- Do not include resources that are **unmaintained, broken, or taken down** (unless historically significant and noted as such)
- Prefer **primary sources** over aggregated or reposted content
- For tweets, include the **author handle, a brief quote, and key metrics** (views/likes) when available

## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/1/code_of_conduct/). By participating, you are expected to uphold this code. Please report unacceptable behavior by opening an issue.

### Our Pledge

We pledge to make participation in our project a harassment-free experience for everyone, regardless of age, body size, disability, ethnicity, gender identity and expression, level of experience, nationality, personal appearance, race, religion, or sexual identity and orientation.

### Our Standards

Examples of behavior that contributes to a positive environment:

- Using welcoming and inclusive language
- Being respectful of differing viewpoints and experiences
- Gracefully accepting constructive criticism
- Focusing on what is best for the community

Examples of unacceptable behavior:

- Trolling, insulting comments, and personal attacks
- Public or private harassment
- Publishing others' private information without explicit permission
